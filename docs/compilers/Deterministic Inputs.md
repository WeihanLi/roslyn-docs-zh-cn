确定性输入
====================

当指定 `/deterministic` 选项时（这是 .NET SDK 中的默认设置），C# 和 VB 编译器是完全确定性的。这意味着"相同的输入"将导致编译器生成逐字节"相同的输出"。

以下被视为编译器的确定性输入：

- 命令行参数的序列（顺序很重要）
- 使用的编译器的确切版本及其部署中包含的文件：引用程序集、rsp 等...
- 当前完整目录路径（您可以将其减少为相对路径；参见 https://github.com/dotnet/roslyn/issues/949）
- 直接或间接显式传递给编译器的所有文件的（二进制）内容，包括
  - 源文件
  - 引用的程序集
  - 引用的模块
  - 资源
  - 强名称密钥文件
  - `@` 响应文件
  - 分析器
  - 生成器
  - 规则集
  - 分析器和生成器可能使用的"附加文件"
- 如果未指定 `/preferreduilang`，则为当前区域性（用于生成诊断和异常消息的语言）。
- 如果未指定 `/codepage` 并且任何输入源文件没有 BOM 且不是 UTF-8 编码，则为当前操作系统代码页。
- 编译器搜索路径上文件的存在、不存在和内容（例如，由 `/lib` 或 `/recurse` 指定）
- 编译器运行的 CLR 平台：
  - 在某些平台上，为常量折叠执行的 `double` 算术结果可能使用过多的精度。
  - 编译器使用平台提供的 Unicode 表。
- CLR 用于实现压缩的 zlib 库版本（当使用 `/embed` 或 `/debug:embedded` 时）。
- `%LIBPATH%` 的值，因为如果不完全限定，它可能会影响引用发现以及运行时如何处理分析器/生成器依赖项加载。
- 源文件的完整路径，尽管可以使用 `/pathmap` 在不同根目录中相同代码的编译之间规范化此路径。

目前，编译器还依赖于一天中的时间和 GUID 的随机数，因此除非您指定 `/deterministic`，否则它不是确定性的。
