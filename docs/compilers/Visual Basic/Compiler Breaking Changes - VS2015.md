**本文档列出了 Roslyn（VS2015 及更高版本）相对于本机 VB 编译器（VS2013 及以前版本）的已知重大更改。**

### VB（与语言规范或本机编译器的偏差）

此列表追踪 Roslyn 编译器有意偏离语言规范或以前版本的已知情况。它既包括重大更改，也包括使得以前在语言规范或本机编译器中不被允许的情况成为可能的更改。

- [ ] 当将输出扩展名指定为已知输出类型之一（例如 /out:foo.dll），但指定不同的目标类型（例如 /t:exe）时，编译器现在将保留指定的扩展名，不会添加目标扩展名（例如 foo.dll.exe）。有关示例和详情，请参见 [#13681](https://github.com/dotnet/roslyn/issues/13681)。
- [ ] 现在允许在隐式行续行之后使用注释。
- [ ] 重大更改：Unicode 字符 U+30FB 和 U+FF65 不再解析为有效标识符字符（Bug 3246）。
- [ ] 重大更改：Unicode 字符转义序列不再在预处理器指令关键字中解析（Bug 1433）。
- [ ] 重大更改：受保护的泛型类型的一个构造的成员现在可以从该类型的其他构造中访问；重载解析可能会改变（Bug 4107）。
- [ ] 重大更改：Unicode 字符 U+180E（蒙古语元音分隔符）不再解析为有效标识符字符（Bug 4189）。

### VB 中常量表达式的规则

根据 @AlekseyTs 的说法，"Dev10 在某些情况下即使这些表达式不会被求值也认为它们在常量表达式上下文中无效。Lambda、查询表达式和可能的对象初始化器都属于这一类。"

这些规则在 Roslyn 中得到了放宽。如果形式为 `expr.ID` 且 ID 是该表达式类型的常量成员，则允许常量表达式。该表达式不会被求值，并且会发出警告。对 expr 的形式没有限制，只需要它在语法和语义上有效。
