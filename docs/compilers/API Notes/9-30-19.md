## 2019 年 9 月 30 日 API 审查笔记

### 已审查的更改
起始提交：`12b125030c34e340086b7fa192fb9426aaddf054`

### 笔记

关于 AnalyzerConfig 的移除，我们表示接受此更改，因为该 API 刚刚发布，且并非有意发布。
我们不希望在未来维护该 API 形态，愿意为了现在将其移除而承担引入破坏的风险，趁着许多人还未对该 API 表面产生强依赖之前。

`ErrorLogPath` 从 `CommandLineArguments` 中移除，移至 `ErrorLogOptions.Path`
 - 我们应该保留此 API。
 - 直接转发到旧实现即可。
 - @333fred 将提交一个 PR。

TextDocument
 - 我们是否不应该移除这个构造函数？这是一个二进制层面的破坏性更改。
 - 这个空构造函数无法被使用。
 - 我们将保留此更改。

Workspace
 - `CanApplyParseOptionChange` 从 `virtual protected` 改为 `virtual public` 既是源代码层面也是二进制层面的破坏性更改。
 - 应考虑在 `Workspace` 上添加一个新的非虚公有方法，并将其转发给此 API。
 - @jasonmalinowski 将负责此修复。

SyntaxFactory.AnonymousMethodExpression
 - 添加了带多个参数的重载。
 - 属于 https://github.com/dotnet/roslyn/pull/37674 的一部分，该 PR 使 API 与 `MethodBodySyntax` 保持一致。
   此更改没有问题。

Formatter.OrganizeUsings
 - 是否应将 `CancellationToken` 设为可选参数？
 - 需要与框架跟进当前的指南是什么。
 - 暂时将其设为默认值。
 - @jasonmalinowski 跟进此问题。
