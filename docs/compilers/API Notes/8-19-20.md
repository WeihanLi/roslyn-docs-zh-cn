### 笔记

#### `SkipAnalyzers` 是否也应该跳过诊断抑制器？

我们添加了一个命令行开关，允许用户在运行编译器时跳过分析器。这主要是出于性能考虑，让客户能够完成我们在 Roslyn 构建中一直在做的事情，而无需通过 MSBuild 技巧在构建前移除分析器。它同样适用于源代码生成器，后者通过同一属性处理。然而，有人担忧在此场景下抑制器不会运行，这可能导致某些构建失败。尽管这是一个真实的问题，但反方观点指出，运行抑制器基本上会使该标志本来要带来的所有性能收益付之东流。此议题需要进一步讨论。

#### `GeneratedSourceResult.HintName` 描述性可能不够充分

`GeneratedSourceResult` 上的 `HintName` 属性描述性不强，未能反映出对该属性稳定性的期望。应将其重命名为能够表明它应保持稳定（或至少确定性）的名称。

#### `GeneratorDriver.RunFullGeneration` 在新 API 面前似乎命名不当

我们现在有了 `GeneratorDriver.RunGenerators` 方法。有了这个方法，`RunFullGeneration` API 的命名似乎不太合适。如果它仍然必要，可以考虑将其命名为类似 `RunGenerationAndCreateCompilation` 的名称。

#### 移除 `SymbolDisplayMiscellaneousOptions.UseExplicitManagedCallingConventionSpecifier`

我们目前没有计划在 IDE 中使用这个选项。可以先移除，等到有 API 请求时再重新添加。

#### `SyntaxTreeOptionsProvider.IsGenerated` 应使用枚举，而非三值状态

相比使用可空布尔值（不能很好地描述 `null` 的含义），此处应使用具名枚举以提高清晰度。

#### `SyntaxTreeOptionsProvider` 的方法应接受 `CancellationToken`

我们预计此类的实现在方法调用时进行计算是合理的，以便 IDE 能够做到惰性求值。因此，这些方法应接受 `CancellationToken`，允许 IDE 在必要时取消工作。

#### `IMethodSymbol.CallingConventionTypes`

应重命名为 `UnmanagedCallingConventionTypes`，以与语法保持一致。
