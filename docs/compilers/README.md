# Roslyn 编译器

## 规范

编译器规范详细说明了 Roslyn VB 和 C# 编译器支持的（和半支持的）表面区域。这包括

0. 命令行开关及其含义
1. 与先前版本编译器的破坏性变更
2. （故意）与规范相反的编译器行为
3. 语言规范未描述的编译器功能
    1. COM 特定和其他 Microsoft 特定功能
    2. 影响编译器行为的"众所周知的"属性
    3. "规则集"文件语法和语义
4. 为 C# 和 VB 之间的互操作性而包含的功能，例如
    1. 从 C# 使用命名索引器
5. 编译器行为与语言规范不同的地方
6. 限制（例如标识符长度）
7. 每个版本的语言更改历史

语言规范本身不包含在此处。

## 平台

编译器在以下配置中得到官方支持：

- .NET Framework：x86、x64 和 ARM64
- .NET Core：
  - 操作系统：Windows、macOS、Linux
  - 架构：x86、x64 和 ARM64
- Mono Core：x86 和 x64

编译器接受其他平台（如大端架构）的贡献，但不在那里正式支持编译器。
