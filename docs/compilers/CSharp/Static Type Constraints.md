# 静态类型约束

Roslyn C# 编译器未实现 C# 语言规范所要求的所有关于静态类型的约束。

本机编译器在类型参数被推断为静态类型时会给出错误，例如在如下调用中：

```cs
M(default(StaticType))
```

现在这只是可能的，因为本机编译器没有强制执行不能在 `default` 表达式中使用静态类型的限制。这是第一个 bug（#344），由于兼容性原因，不会在 Roslyn 中修复。

现在推断的类型参数是静态类型，本机编译器会为此给出错误。虽然语言规范说将静态类型用作泛型类型参数是非法的，但不清楚这是否也适用于推断的类型参数。无论如何，由于推断的类型参数是静态类型，本机编译器在上述调用中会给出错误。

Roslyn 仅在绑定显式类型参数时实现了约束，因此 Roslyn 没有诊断此情况。这是第二个 bug（#345）。通过将检查从绑定类型参数的地方移到检查类型参数对类型参数约束的地方，在 Roslyn 中"修复"了这个问题。

作为另一个独立问题，本机编译器在动态调用中将静态类型用作类型参数时不会给出错误，而 Roslyn 会给出错误：

```cs
((dynamic)3).M<StaticType>()
```

这是第三个 bug（#511），虽然 Roslyn 的行为符合规范，但这是对本机编译器的重大更改。但事实证明，这个兼容性问题被上述修复解决了，因为动态调用中的类型参数不会针对任何类型参数约束进行检查。因此，Roslyn 现在无法诊断此情况（即未实现所需的语言规则），但至少在这里与本机编译器兼容。
