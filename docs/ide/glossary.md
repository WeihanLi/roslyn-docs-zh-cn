# IDE 术语表

* **IntelliSense** - 一系列功能的总称，包括**补全**、**签名帮助**、**快速信息**和**智能标签**。

* **补全**（又称**自动完成**或**语句补全**）- 在当前位置插入编辑器中的可能项的下拉列表。通常在键入或按 `Ctrl+J` 或 `Ctrl+Space` 时自动触发。可以处于两种模式之一，*补全模式*或*建议模式*。在*补全模式*（默认）中，键入任何不属于当前选定条目的字符会"提交"当前条目并插入其文本。*建议模式*补全显示一个"构建器"，当前项是"软选中"的（以轮廓显示，而不是完全选中）。在建议模式下，键入不在列表中的字符不会提交项，要提交必须使用 tab/enter/双击显式执行。您可以通过按 `Ctrl+Alt+Space` 在*补全模式*和*建议模式*之间切换。在不明确您是插入新标识符还是引用现有标识符的情况下，IDE 通常强制*建议模式*，以便在尝试声明新标识符时不会意外提交现有项（例如，在期望委托的上下文中，强制建议模式以允许键入 lambda 参数名称）。

* **补全构建器** - 在*建议模式*下，补全在列表顶部显示一个额外的条目，如果编辑器中没有文本，则包含显示可以出现的声明类型的水印，否则显示与当前键入文本匹配的条目。这旨在表示补全不会提交到列表中的另一项，而是会保留您当前的文本，除非强制提交（使用 tab/enter）。

* **签名帮助**（又称**参数帮助**）- 显示键入方法调用或类型/方法的泛型参数时可能的重载的工具提示。通常使用 `Ctrl+K, Ctrl+P` 或 `Ctrl+Shift+Space` 调用。

* **快速信息** - 悬停在标识符上时显示有关它的信息的工具提示。也可以使用 `Ctrl+K, Ctrl+I` 调用。

* **智能标签** - VS 2005 到 VS 2013 中出现在标识符下方的小蓝色或红色方块，指示 IDE 可以帮助执行操作。在 VS 2015 中被灯泡取代。

* **灯泡** - 指示 IDE 可以帮助执行某些操作，以左边距中的灯泡图标或快速信息工具提示的形式出现。通常使用 `Ctrl+.` 调用。菜单项通常称为`代码操作`，有两种类型 - 用于诊断的`快速修复`或`重构`。快速修复在光标位于带有诊断的行上时显示灯泡，而重构在调用 `Ctrl+.` 时按需显示。

* **边距** - 核心编辑器窗口周围的保留空间。大多数边距出现在编辑器的左侧，例如：行号、更改跟踪、大纲、断点、灯泡等。

* **大纲** - 允许折叠代码区域的功能。

* **分类** - 语法高亮发生的过程。在 Roslyn 中，有两种类型的分类 - *语法*分类同步发生（尽管可能基于过时的语法树），*语义*分类在轻微延迟后发生，负责为类型名称、`var`、`dynamic` 等着色。

* **波浪线** - 具有某种诊断的标识符下的波浪下划线。通常错误为红色，警告为绿色，编辑并继续粗糙编辑为紫色。

* **导航栏** - 代码编辑器顶部的三个下拉列表，允许您选择项目上下文、类型和成员。

* **标记** - VS 编辑器 API 中的一般术语，允许像 Roslyn 这样的扩展将范围集合返回给编辑器。许多 IDE 功能都建立在标记之上：分类、大纲、波浪线、行分隔符等。

* **装饰** - VS 编辑器 API 中的一般术语，用于向编辑器添加视觉元素，通常是响应标签的存在。C# 和 VB 看到的许多装饰实际上是在核心编辑器中实现的（波浪线、分类、大纲等），但有一些在 Roslyn 中（建议、行分隔符）。
