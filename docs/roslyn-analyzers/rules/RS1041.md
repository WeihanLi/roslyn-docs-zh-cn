## RS1041：编译器扩展应在以 netstandard2.0 为目标框架的程序集中实现

实现编译器扩展点的类型，应仅声明在以 netstandard2.0 为目标框架的程序集中。更具体的目标框架仅在部分受支持的编译场景中可用，因此以其为目标可能导致功能行为不可预测。

|项目|值|
|-|-|
|类别|MicrosoftCodeAnalysisCorrectness|
|已启用|True|
|严重性|Warning|
|CodeFix|False|
---

此规则有助于确保编译器扩展（例如分析器和源生成器）在所有编译场景中都能正确加载。根据编译器的调用方式，编译器可能在 .NET Framework 或 .NET 下执行，编译器扩展应在两种环境中保持一致的行为。通过以 netstandard2.0 为目标框架，可以确保编译器扩展与两种执行环境兼容。

### 编译器扩展点

此分析器检查以下编译器扩展点：

* `DiagnosticAnalyzer`
* `DiagnosticSuppressor`
* `ISourceGenerator`
* `IIncrementalGenerator`
