## RS1038：编译器扩展应在仅包含编译器提供引用的程序集中实现

实现编译器扩展点的类型，不应声明在包含并非所有编译场景都提供的程序集引用的程序集中。否则可能导致该功能行为不可预测。

|项目|值|
|-|-|
|类别|MicrosoftCodeAnalysisCorrectness|
|已启用|True|
|严重性|Warning|
|CodeFix|False|
---

此规则有助于确保编译器扩展（例如分析器和源生成器）在所有编译场景中都能正确加载。根据编译器的调用方式，某些程序集在构建时可能不存在，尝试引用它们将导致阻止编译器扩展加载的异常。RS1038 是针对此场景最严格、性能最优的验证方式。

RS1038 默认启用，除非已在 **.globalconfig** 中按 [RS1022](RS1022.md) 中的描述手动启用宽松验证。

### 编译器功能引用规则

* 支持 C# 代码的编译器功能，应仅引用 NuGet 包 **Microsoft.CodeAnalysis.Common** 和/或 **Microsoft.CodeAnalysis.CSharp**
* 支持 Visual Basic 代码的编译器功能，应仅引用 **Microsoft.CodeAnalysis.Common** 和/或 **Microsoft.CodeAnalysis.VisualBasic**
* 同时支持 C# 和 Visual Basic 代码的编译器功能，应仅引用 **Microsoft.CodeAnalysis.Common**
* 编译器功能不应在包含对 **Microsoft.CodeAnalysis.Workspaces.Common** 引用的程序集中实现

> **注意**
>
> 此分析器仅检查对核心 Roslyn 程序集的引用。具有其他依赖项的编译器扩展
> 可能面临超出此分析器范围的限制和/或打包要求。

### 编译器扩展点

此分析器检查以下编译器扩展点：

* `DiagnosticAnalyzer`
* `DiagnosticSuppressor`
* `ISourceGenerator`
* `IIncrementalGenerator`

### 其他扩展点

Roslyn 提供的某些扩展点是 IDE 扩展（例如代码修复和补全提供程序）。这些功能可以与编译器功能打包在同一个包中，但应在各自独立的程序集中实现，因为它们需要引用非编译器包 **Microsoft.CodeAnalysis.Workspaces.Common**。
