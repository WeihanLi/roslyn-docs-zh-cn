# IDE 团队设计流程

## 讨论 / 问题

目标是让贡献者能够创建和讨论功能建议，而无需事先把一切都想清楚。欢迎提交新的 [GitHub issue](https://github.com/dotnet/roslyn/issues/new?assignees=&labels=Area-IDE%2CFeature+Request&template=feature-suggestion.md) 或发起[讨论](https://github.com/dotnet/roslyn/discussions/new)。这些对话可以按自己的节奏进行，直到贡献者准备好提出更正式的提案。

## 提案

### 添加提案

要让功能提案获得审查，需要创建一个 GitHub issue，其描述中包含以下内容。如果一开始没有这些内容，在提交讨论之前最终需要以评论的形式补充到 issue 中。

- **摘要**：关于本提案的简要说明。
- **背景与动机**：我们正在解决什么问题？您是在什么情况下遇到它的？
- **提议的功能**：建议的解决方案如何解决该问题？
- **替代设计**：您是否考虑过其他选项？与其他编辑器中的功能相比如何？

添加功能详情后，请@一位 Roslyn-IDE 团队成员，他们将为您的 GitHub issue 添加"[Feature Request](https://github.com/dotnet/roslyn/issues?=is%3Aissue+is%3Aopen+sort%3Aupdated-desc+label%3A%22Feature+Request%22)"标签。

### 由 IDE 团队认领 issue

[Roslyn-IDE](https://github.com/orgs/dotnet/teams/roslyn-ide) 的成员需要决定"认领"该功能提案，并通过将其添加到 [IDE Design Review](https://github.com/dotnet/roslyn/projects/40) 项目的"On Deck"选项卡中来提交讨论。在幕后，IDE 团队通常每周开一次会来讨论这些 issue。
