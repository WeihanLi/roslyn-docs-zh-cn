Out 变量声明
=========================

*out 变量声明*特性允许在将变量作为 `out` 参数传递的位置声明该变量。

```antlr
argument_value
    : 'out' type identifier
    | ...
    ;
```

以这种方式声明的变量称为 *out 变量*。
你可以使用上下文关键字 `var` 作为变量的类型。
其作用域与通过模式匹配引入的*模式变量*相同。

根据语言规范（第 7.6.7 节，元素访问），元素访问（索引表达式）的参数列表不包含 ref 或 out 参数。
但在各种场景下编译器允许这样做，例如在接受 `out` 的元数据中声明的索引器。

在由局部变量声明引入的局部变量的作用域内，在文本位置上先于其声明处引用该局部变量是编译时错误。

在同一个参数列表中，直接引用隐式类型（§8.5.1）的 out 变量也是一个错误。

重载解析的修改如下：

添加了一种新的转换：

> 从表达式到隐式类型的 out 变量声明存在*来自表达式的转换*，适用于所有类型。

同时：

> 显式类型的 out 变量参数的类型是声明的类型。

以及：

> 隐式类型的 out 变量参数没有类型。

当参数是隐式类型的 out 变量声明时，两种来自表达式的转换都不更优。（这需要织入到规范的形式中）

隐式类型的 out 变量的类型是方法签名中对应参数的类型。

新增了语法节点 `DeclarationExpressionSyntax`，用于表示 out var 参数中的声明。
