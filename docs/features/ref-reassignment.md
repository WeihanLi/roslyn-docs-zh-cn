Ref 局部变量和参数的重新赋值
====================================

在 C# 7.2 中，`ref` 局部变量和参数无法被赋予新的引用，只能通过赋值来更改底层存储的值。

C# 7.3 引入了一种新的表达式类型——ref 赋值表达式。

语法如下：

```
ref_assignment
    : identifier '=' 'ref' expression
```

*ref_assignment* 左侧的标识符必须是 `ref` 或 `ref readonly` 局部变量，或者是 `in`、`ref` 或 `out` 参数。Ref 赋值允许表示对存储位置的引用的 ref 局部变量或参数被赋予不同的引用。这不包括 `this` 引用，即使在 struct 中也不行。

右侧的表达式必须具有存储位置，即它必须是"l 值"。该 l 值必须与被 ref 赋值的变量"ref 兼容"，ref 兼容的含义是：

* 左侧类型和右侧类型之间必须存在恒等转换。
* 可写的 ref 变量不能被赋予只读的 l 值。只读的 ref 变量可以被赋予只读或可写的 l 值。

ref 赋值表达式的结果本身是一个"l 值"，且当且仅当赋值左侧是只读 ref 变量时，所得到的存储位置是只读的。

右侧的存储位置必须在 ref 赋值表达式中使用之前已明确赋值。此外，`out` 参数所引用的存储位置必须在该 `out` 参数被 ref 重新赋值到新引用之前已明确赋值。

`ref` 局部变量或参数的生存期在声明时固定，不能通过 ref 重新赋值来更改。右侧表达式的生存期要求与变量赋值中右侧表达式的要求相同，与 `Span<T>` 赋值的生存期要求一致。
